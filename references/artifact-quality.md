# Artifact Quality

Use this checklist before presenting an HTML artifact.

## Hard Requirements

- Single file with inline CSS, JS, and SVG.
- No external network dependencies by default:
  - no `<script src=...>`
  - no `<link rel=stylesheet href=...>`
  - no `@import`
  - no remote image URLs
  - no analytics or external fetches
- Opens through `file://` without a server.
- Includes viewport meta.
- Uses real content from the task, not placeholder examples.
- Provides a clear first viewport: title, purpose, source/provenance, primary action.
- Uses semantic HTML and a shape appropriate to the job.
- Avoids body-level horizontal scroll at narrow widths.

## HTML-Native Features

Static report artifacts should use at least two. Interactive/editor artifacts should use at least three.

- comparison grid
- timeline
- inline SVG diagram or chart
- rendered tokens/components
- sortable or filterable table
- live search/highlight
- tabs/details/slide navigation
- sliders, toggles, drag-and-drop, editable fields
- copy/export controls
- cross-highlighting or click-to-focus

## Export Rules

If the artifact lets the user manipulate state:

- Include a visible primary export button near the working area.
- Export structured text: prompt, Markdown, JSON, CSV, or diff.
- Use clipboard with a visible textarea fallback.
- Export only the relevant state or delta, not the entire HTML file.
- Label export scope: single, selected, bucket, changed-only, or whole.

## Visual Discipline

Prefer the Thariq example vocabulary:

- warm ivory/paper/slate/clay palette
- serif display, sans body, mono metadata/code
- restrained cards, clear borders, tables, SVG, timelines
- purposeful spacing, not decorative whitespace

Avoid:

- gradient-heavy hero pages
- glassmorphism for documents
- emoji-heavy headings
- generic SaaS stat cards when stats are not the subject
- decorative controls
- icon-library dependency
- "Generated by AI" footers

## Security And Privacy

Before sharing outside the local machine, scan for:

- secrets, tokens, API keys, passwords
- private file paths like `/Users/`, `/home/`, `~/`
- internal hostnames and LAN IPs
- personal phone numbers, addresses, account numbers
- private source snippets that should not leave the repo
- external assets that leak opens to third parties

When anything sensitive appears, report it and ask before publishing.
